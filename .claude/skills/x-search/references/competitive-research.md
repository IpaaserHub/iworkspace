# 競合リサーチマニュアル

X上の投稿を活用して、競合企業・競合プロダクトを徹底的に調査するためのリファレンス。

## 概要

このマニュアルは、ユーザーが指定した競合について多角的にX上の情報を収集・分析するためのフローを定義する。単発の検索ではなく、段階的に深掘りしていくリサーチプロセス。

## フロー

### Phase 1: 競合の特定

ユーザーが競合名を指定している場合はそのまま進む。指定がない場合は AskUserQuestion で確認する。

```
AskUserQuestion:
  "競合リサーチを行います。以下を教えてください：
   - 調べたい競合企業/プロダクト名（複数可）
   - 競合のXアカウント（わかれば）
   - 自社プロダクトとの比較観点（あれば）"
```

確認すべき情報：
- 競合の企業名・プロダクト名
- Xの公式アカウント（わかれば）
- 業界・カテゴリ
- 特に気になっている観点（価格、機能、評判など）

### Phase 2: ラフスキャン（コンテキスト把握）

競合が特定できたら、まず**広く浅く**検索して全体像をつかむ。以下の検索を**並行で最大4つ**実行する。

#### 検索パターン

1. **公式アカウントの直近投稿**
   - 競合の公式アカウントの最近の投稿を取得
   - 何を発信しているか、どんなトーンか、頻度はどうか

2. **プロダクト名・企業名でのキーワード検索**
   - ユーザーからの反応・評判・言及を収集
   - ポジティブ/ネガティブの傾向を把握

3. **競合の主要メンバー（CEO、創業者など）の投稿**
   - 戦略的な発言、ロードマップのヒント、業界への見解

4. **競合 vs 自社（または他社）の比較言及**
   - 「A vs B」「AからBに乗り換えた」などの比較投稿

```
# 並行実行の例（最大4つ）
Agent tool 1: "X検索: {競合名}公式アカウント直近投稿"
Agent tool 2: "X検索: {競合名}ユーザー評判"
Agent tool 3: "X検索: {競合名}創業者/CEO投稿"
Agent tool 4: "X検索: {競合名} vs 比較言及"
```

### Phase 3: ユーザーへの中間報告 & 深掘り方向の確認

ラフスキャンの結果をユーザーに共有し、**次に何を深掘りするか**を確認する。これが最も重要なステップ。

```
AskUserQuestion:
  "競合 {名前} のX上の動向をざっと調べました。概要は以下の通りです：

   📌 公式アカウントの傾向:
   - {要約: 例）新機能のリリース告知が多い、週3-4回投稿}

   📌 ユーザーの反応:
   - {要約: 例）UIの使いやすさは評価されているが、価格への不満が散見}

   📌 主要メンバーの発言:
   - {要約: 例）CEOがエンタープライズ展開を示唆する投稿}

   📌 比較言及:
   - {要約: 例）自社プロダクトとの比較では機能面で言及されることが多い}

   さらに深掘りしたい方向はどれですか？"

  options:
    - "ユーザーの不満・ペインポイントを詳しく"
    - "競合の新機能・ロードマップ情報を詳しく"
    - "競合と自社の比較をもっと詳しく"
    - "その他（具体的に指定）"
```

### Phase 4: ディープダイブ（深掘り検索）

ユーザーが選んだ方向に応じて、さらに具体的な検索を実行する。

#### パターン別の深掘り検索

**ユーザーの不満・ペインポイント**
- `"{競合名} 使いにくい / 不満 / バグ / 解約"` などネガティブキーワード
- `"{競合名} から乗り換え / やめた / 代替"` など離脱系キーワード
- 期間を直近1-3ヶ月に絞って傾向変化を確認

**新機能・ロードマップ**
- 公式アカウントの投稿をリリース告知に絞って検索
- `"{競合名} 新機能 / アップデート / リリース / ベータ"` で検索
- 競合のエンジニアやPMのアカウントがわかればそちらも検索

**比較・ポジショニング**
- `"{競合名} vs {自社名}"` の直接比較
- `"{競合名} vs"` で他社との比較も網羅
- `"{競合名} 比較 / レビュー / 選んだ理由"` で意思決定文脈を収集

**その他のユーザー指定方向**
- ユーザーの指定に応じて柔軟に検索クエリを設計

### Phase 5: 最終レポート

すべての検索結果を統合し、以下の構成でレポートをまとめる。

```markdown
# 競合リサーチレポート: {競合名}
## 調査日: {日付}

### エグゼクティブサマリー
- 3-5行で競合の現状と注目ポイントを要約

### 1. 公式発信の傾向
- 投稿頻度、トーン、主なテーマ

### 2. ユーザーからの評価
- ポジティブな声
- ネガティブな声・ペインポイント
- 全体的なセンチメント

### 3. 戦略的動向
- 新機能・プロダクトの方向性
- 主要メンバーの発言から読み取れる戦略

### 4. 比較ポジショニング
- 自社との比較における強み/弱み
- ユーザーが比較時に重視しているポイント

### 5. アクションアイテム（示唆）
- この調査結果から考えられる打ち手や注意点
```

レポートは `/Users/shotanakayama/projects/iworkspace/workspace/x-results/reports/` に保存する。

## 注意事項

- 各Phaseで検索の並行実行は**最大4つまで**
- Phase 3の中間報告は省略しない（ユーザーの関心に沿った深掘りが目的）
- 検索言語は競合のターゲット市場に合わせる（日本企業なら日本語中心、グローバル企業なら英語も含む）
- 同一アカウントの重複取得を避け、新しい情報を優先する
